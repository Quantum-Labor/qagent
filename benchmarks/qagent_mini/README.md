# qagent-mini-50

A hand-crafted, seeded benchmark for select-`k`-from-`N` tool selection. Its
purpose is to measure interaction-aware selection quality: every task has tool
relevances **and** pairwise synergies/redundancies, so a greedy top-k baseline is
provably insufficient and an interaction-aware solver (brute force, QAOA) is
needed.

## Contents

- **50 tasks**: 25 "small" (`N = 8`, `k = 3`) + 25 "full" (`N = 16`, `k = 5`).
- Generated by [`scripts/_gen_qagent_mini.py`](../../scripts/_gen_qagent_mini.py)
  with a fixed seed (`20260608`), so the dataset is fully reproducible.
- The dataset is its own ground truth: each task's `optimal_subset` /
  `optimal_score` is filled in by the brute-force oracle
  (`qagent.oracle.bruteforce`).

## Schema (`dataset.json`, one object per task)

| field | meaning |
| --- | --- |
| `id` | task id, e.g. `s01` (small) / `f01` (full) |
| `description` | short text description of the task |
| `tier` | `small` or `full` |
| `n_tools` | number of candidate tools `N` |
| `k` | number of tools to select |
| `tools` | list of `{index, name}` tool metadata |
| `weights` | length-`N` standalone relevances `w_i` |
| `synergy` | `N x N` symmetric matrix of pair interactions `J_ij` (zero diagonal) |
| `optimal_subset` | brute-force optimal selection (sorted indices) |
| `optimal_score` | score of the optimal subset |

`score(S) = sum_{i in S} w_i + sum_{i<j in S} synergy[i][j]`.

## Honesty note

The tasks are **synthetic** — random relevances and a sparse random synergy
matrix (about 40% of pairs interact). No real tool metadata or real-world utility
is claimed. The benchmark measures whether a solver accounts for tool
interactions, not which tools are useful in practice.

## Reproduce

```bash
python scripts/_gen_qagent_mini.py      # regenerate dataset.json (deterministic)
python scripts/run_benchmarks.py        # brute-force vs greedy vs QAOA accuracy
```
